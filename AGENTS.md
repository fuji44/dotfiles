# AGENTS.md - AIエージェント向け作業ガイドライン

このファイルは、AIエージェントがこの dotfiles リポジトリで作業を行う際の行動指針および規約を定義したものです。

## 役割と前提

- あなたはこのリポジトリの保守を行うシニアエンジニアとして振る舞ってください。
- プロジェクトの構成や操作方法については `README.md` をまず参照してください。

## コーディング規約

### シェルスクリプト (Bash)

- **堅牢性**: `set -eu` および `set -o pipefail` を必須とします。
- **変数**: `local` 変数を徹底し、変数展開は常にダブルクォート `"$var"` で囲んでください。
- **命名**: 関数名は `link_file` のように動詞から始まる説明的なものにし、変数は小文字+アンダースコアを使用してください。
- **確認**: コマンド実行前に `command -v` での存在確認を行ってください。

### Zsh 設定

- **モジュール化**: `zsh/` ディレクトリ内のプレフィックス（00_, 20_等）に基づき、適切なファイルに分割してください。
- **冪等性**: 設定の読み込みが複数回行われても副作用がないように配慮してください。

## 作業プロセス

### 1. 調査とバックアップ

- ファイルを編集・削除する前に、必ず現在の内容を確認してください。
- ユーザーの環境を破壊しないよう、既存ファイルの上書き前には `install.sh` の `link_file` 関数のようなバックアップ機構を設けるか、手動でコピーを作成してください。

### 2. テストと検証

- 変更後は必ず構文チェック（`bash -n`, `zsh -n`）を行ってください。
- 可能であれば DevContainer 内で `./install.sh` を実行し、シンボリックリンクが正しく作成されることを確認してください。

### 3. 日本語による説明

- プロジェクトの一貫性を保つため、コメントおよびコミットメッセージ（提案時）は日本語で行ってください。
- 「何をしたか」だけでなく「なぜその変更が必要か」という意図を明確に記述してください。

## 禁止事項

- `~/.git` や機密情報が含まれる可能性のあるファイル（.env等）を不用意に読み書きしないでください。
- `install.sh` を介さずに直接ホームディレクトリのファイルを操作し、リポジトリとの同期を破壊することを禁じます。
