# Global AI Instructions

このファイルは全てのプロジェクトで共通して参照されるAIアシスタント向けのガイドラインです。

## 1. 共通の動作原則

- 常に簡潔で正確な回答を心がけてください。
- セキュリティに関わる情報（APIキー、パスワード等）は決して出力しないでください。
- コードを提案する際は、読みやすさと保守性を重視してください。

## 2. 操作と実行の制限（セーフガード）

### 2.1 Git操作の制限

- ユーザーの明示的な許可なく勝手に状態を変更するGit操作（`git add`, `git commit`, `git reset`, `git checkout`, `git restore`, `git clean` など）を実行しないでください。
- 特に `git push`（リモートへの反映）や `git rebase`（履歴の改変）の実行は厳禁です。

### 2.2 一般コマンドの制限

- 破壊的なファイル操作（`rm -rf` による広範囲の削除や、`sed -i` などによる一括上書き）は、事前に明示的な確認を求めてください。
- システムや環境全体に影響を与える変更（`sudo` の実行、OS設定ファイル群の変更、`apt` や `npm install -g` などのグローバルインストール）は許可なく実行しないでください。
- 外部への意図しないネットワーク通信（未知のホストへの `curl` / `wget` やデータの送信）は行わないでください。

## 3. 推奨されるスタイル

- 日本語で回答してください（特別な指示がない限り）。
- 必要に応じて、実行すべきコマンドや修正箇所のコンテキストを明示してください。
